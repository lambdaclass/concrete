Concrete

Systems code that carries its evidence.

Concrete is a no-GC systems language with linear types, explicit capabilities, and source-level evidence. The end goal is simple: write the claim next to the code, let the compiler generate the obligations, and audit how each one was discharged.

Target language

The claim lives in the file that runs.

The current project already has proof registries, drift gates, audit output, assumptions, oracles, and Lean-backed examples. The next language step is to move the proof surface into Concrete source: specs, contracts, loop invariants, ghost values, and discharge hints.

The runtime language stays small. These annotations are compile-time evidence. They erase before codegen.

Lean + automated leaves Loop structure in Lean; byte facts by decision procedure 
spec fn diff_prefix(a: [u8; 16], b: [u8; 16], n: i32) -> u8;
contract SameTag(a: [u8; 16], b: [u8; 16], r: Int);

#[ensures(SameTag(a, b, result))]
#[prove_by(lean)]
fn ct_compare(a: [u8; 16], b: [u8; 16]) -> Int {
    let mut diff: u8 = 0;
    let mut i: i32 = 0;

    #[invariant(0 <= i && i <= 16)]
    #[invariant(diff == diff_prefix(a, b, i))]
    #[variant(16 - i)]
    while i < 16 {
        set diff = diff | (a[i] ^ b[i]);
        set i = i + 1;
    }

    if diff == 0 { return 1; }
    return 0;
}

Audit Workflow

The compiler refuses to let evidence silently lie.

01

Write source

Capabilities, ownership, and trusted boundaries are visible in code.

02

Attach evidence

Lean theorems, oracles, policies, assumptions, and snapshots bind to fingerprints.

03

Audit facts

concrete audit separates proved, tested, enforced, assumed, and trusted.

04

Catch drift

Authority widening, proof/spec drift, and evidence weakening fail normal gates.

$ concrete audit examples/constant_time_tag/src/main.con

proof:
  ct_compare_same_tag_correct: proved_by_lean
  coverage: one_direction

authority:
  capabilities: none
  allocation: none
  trusted code: none

assumptions:
  machine-level constant-time: assumed
  backend timing preservation: trusted

Concrete does not collapse all evidence into one green badge. It separates what is Lean-proved, compiler-enforced, oracle-tested, assumed, and trusted.

What Concrete Does Not Claim Yet

Concrete is not a verified compiler today. It does not prove binary correctness, whole-program constant-time behavior, or all source-to-ProofCore extraction rules. Contracts, verification conditions, and SMT assistance are roadmap work, not shipped guarantees.

The current claim is narrower and more useful: the compiler refuses to let evidence silently lie. If authority widens, allocation appears, assumptions change, or a proof drifts from the source, the audit and CI path should make that visible.
