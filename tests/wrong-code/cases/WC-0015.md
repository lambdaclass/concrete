# WC-0015 — int-match arms didn't reject consumption disagreement

**Category:** error-regression
**Status:** fixed
**Discovered:** historical Bug 021

## Symptom

For an integer-pattern `match` where only some arms consume a linear
value, the linearity checker accepted the program — letting a
linear-discipline violation through. Pair with WC-0014 (the
non-rejection-when-all-agree case).

## Expected behavior

`tests/programs/bug_int_match_disagree.con` is **rejected** by the
compiler with diagnostic code `E0209` (match arms disagree on
consumption).

## Regression command

`make test-wrong-code` — manifest entry is `kind = "compile-error"`,
`expected = "E0209"`. The test asserts the compile *fails* with that
code.

## Historical context

Pre-corpus Bug 021. Same checker site as WC-0014 — the integer-
pattern path had the merge wrong in both directions (didn't
propagate, didn't disagree). Fix landed together.
