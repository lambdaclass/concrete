**Category:** miscompile (high-risk boundary, no recorded prior bug)
**Status:** fixed

## Symptom

No bug surfaced; this case exists as adversarial coverage of a
known-fragile layout interaction. Three layout systems collide here:
heap allocation (`Heap<T>`), arrow-deref (`p->field`), and newtype
unwrapping (`.0`). Each individually has had bugs (see WC-0001,
WC-0002, WC-0007); the combined surface is registered to catch
regressions before they reach a user-visible miscompile.
