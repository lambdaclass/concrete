# WC-0011 — enum fields in structs (and Vec<T>)

**Category:** miscompile
**Status:** fixed
**Discovered:** historical Bug 005

## Symptom

Enum-typed fields inside named structs computed wrong alignment,
which broke both standalone struct reads and `Vec<StructWithEnum>`
storage.

## Expected behavior

`tests/programs/bug_enum_in_struct.con` returns `0`.

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus Bug 005. Layout engine fix to handle enum alignment
inside named structs; same fix made `Vec<T>` work for `T` containing
an enum field.
