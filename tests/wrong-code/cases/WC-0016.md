# WC-0016 — print_string / print_int / print_char builtins

**Category:** miscompile
**Status:** fixed
**Discovered:** historical Bug 007

## Symptom

The print builtins did not produce stdout output at landing (or
produced wrong bytes / wrong format).

## Expected behavior

`tests/programs/bug_print_builtins.con` outputs `hello 42\n0` (the
literal "hello ", the integer `42`, a newline via `print_char(10)`,
and then the main wrapper's `0`-and-newline for the int return).

## Regression command

`make test-wrong-code` (manifest `expected = "hello 42\n0"`, multi-
line).

## Historical context

Pre-corpus Bug 007. The intrinsics live in
`Concrete/EmitBuiltins.lean` and are emitted as LLVM functions that
call `printf`. The regression entry catches both formatter bugs
(wrong format string) and missing-emission bugs (intrinsic not
declared at the LLVM level).
