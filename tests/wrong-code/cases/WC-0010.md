# WC-0010 — cross-module struct fields all read offset 0

**Category:** miscompile
**Status:** fixed
**Discovered:** historical (pre-corpus)

## Symptom

All fields of a struct defined in another module read as offset 0:
reading `p.x` and `p.y` returned the same value (whatever was at
offset 0 of the struct's storage). Same struct defined in the
calling module worked correctly.

## Expected behavior

`tests/programs/bug_cross_module_struct_field.con` returns `42` (a
two-field `Point` defined in module `Types`, accessed from `main`).

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus. Layout / field-offset metadata wasn't surviving the
module boundary; the importing module synthesized a layout where all
fields shared offset 0.
