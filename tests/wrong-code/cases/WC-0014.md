# WC-0014 — int-match arms didn't propagate consumption state

**Category:** error-regression
**Status:** fixed
**Discovered:** historical Bug 020

## Symptom

For an integer-pattern `match` where every arm consumes the same
linear value, the linearity checker failed to merge arm consumption
states. The valid program was rejected with "never consumed".

## Expected behavior

`tests/programs/bug_int_match_consume.con` compiles and returns
`0`.

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus Bug 020. Pair with WC-0015 (the disagree case). Original
checker had a coverage gap on integer-pattern arms specifically;
enum-pattern arms had been correct. Fix in
`Concrete/Check.lean`'s match-arm linearity merge.
