# WC-0009 — cross-module &mut borrow consumed instead of borrowed

**Category:** miscompile
**Status:** fixed
**Discovered:** historical (pre-corpus)

## Symptom

Passing `&mut Vec<T>` to a function in another module consumed
(moved) the borrow, making a second call fail with "linear variable
used after move". The same call shape inside a single module worked
correctly.

## Expected behavior

`tests/programs/bug_cross_module_mut_borrow.con` returns `42` (two
sequential `push_val` calls into a cross-module helper).

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus. Fix lived in the cross-module borrow / linearity
plumbing; the bug surfaced because the cross-module path elaborated
the parameter type slightly differently from the in-module path,
losing the `&mut` distinction.
