# WC-0006 — `arr[i] = val` with variable index wrote to wrong location

**Category:** miscompile
**Status:** fixed
**Discovered:** historical Bug 004

## Symptom

`arr[i] = val` where `i` was a variable (not a constant) emitted a
GEP using the wrong element type, so the store landed at the wrong
byte offset.

## Expected behavior

`tests/programs/bug_array_var_index_assign.con` returns `42`.

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus Bug 004. Fix in `Concrete/Lower.lean`'s `arrayIndexAssign`:
GEP must use the array element type for the index multiplier, not a
generic byte offset.
