# WC-0008 — clock_monotonic_ns builtin

**Category:** miscompile
**Status:** fixed
**Discovered:** historical Bug 012

## Symptom

`clock_monotonic_ns()` did not return a positive timestamp at landing
(missing intrinsic / wrong return path).

## Expected behavior

`tests/programs/bug_clock_builtin.con` returns `0` (the test asserts
that the call produced a non-zero timestamp internally and exits with
0 to signal pass; timing-sensitive monotonicity assertions are
deliberately avoided to keep the test stable).

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus Bug 012. The repro is also intrinsic-existence coverage
(if `clock_monotonic_ns` is removed or renamed, the test fails at
compile time with an undefined-function diagnostic).
