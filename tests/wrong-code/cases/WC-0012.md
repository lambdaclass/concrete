# WC-0012 — i32 literal in binop emitted i64 op

**Category:** miscompile
**Status:** fixed
**Discovered:** historical (pre-corpus)

## Symptom

`0 - a` where `a: i32` generated `sub i64 0, %i32_val` — type
mismatch in the LLVM IR. Either crashed in clang or, depending on
optimization level, produced wrong arithmetic.

## Expected behavior

`tests/programs/bug_i32_literal_type.con` returns `42`. The test
checks `0 - 10 == -10` and a few related shapes.

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus. Fix was to propagate type hints into integer literals
inside binary operations, so an i32-typed operand would force the
literal to be i32-typed too.
