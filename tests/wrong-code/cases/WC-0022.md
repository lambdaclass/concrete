# WC-0022 — trait method dispatched through generic bound

**Category:** miscompile (high-risk boundary, recently-shipped feature)
**Status:** fixed
**Discovered:** 66d8736 adversarial sweep

## Symptom

No bug surfaced in this exact program. The case covers a recently
shipped, complex feature: a generic function with a trait bound that
calls a method on the bound type-parameter, expecting the call site
to resolve to the concrete impl.

## Expected behavior

`tests/programs/adversarial/trait_dispatch/trait_method_via_bound.con`
returns `84`.

## Regression command

`make test-wrong-code`.

## Historical context

Adversarial sweep from `66d8736`. Trait dispatch through bounds
involves multiple compiler passes (`Resolve`, `Check`, `Mono`,
`Lower`) cooperating to pick the right concrete impl at the call
site. A regression in any of those passes would surface as either a
compile failure or a wrong-method-called miscompile.
