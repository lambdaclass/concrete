# WC-0017 — borrowing a stack array as `*mut` wrote into a copy

**Category:** miscompile
**Status:** fixed
**Discovered:** historical Bug 018

## Symptom

Borrowing a stack-allocated array as a `*mut u8` pointer (typically
to pass to a trusted FFI function like `memset`) sometimes
materialized a *copy* of the array, so the FFI write landed in
scratch memory and the original stack slot was untouched.

## Expected behavior

`tests/programs/bug_stack_array_borrow_copy.con` returns `42` (the
test calls `memset` through a stack-array borrow and reads back the
written bytes; if the copy bug returns, the read sees zeros instead
of the written value).

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus Bug 018. Fix lived in the borrow-as-raw-pointer
materialization path: the stack slot's address must be taken
directly, never via a load + re-spill (which creates a copy).
