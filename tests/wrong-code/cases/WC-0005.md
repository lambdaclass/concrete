# WC-0005 — array extracted from struct field is not mutable

**Category:** miscompile
**Status:** fixed
**Discovered:** historical (pre-corpus)

## Symptom

```
let mut d: [i32; 4] = b.data;
d[0] = 99;
return d[0];      // returned 0 instead of 99
```

Direct array init worked (`let mut d = [0; 4]; d[0] = 99;` returns 99),
but copying an array out of a struct field and then mutating did not
materialize the mutation. The compiled binary returned the original
value.

## Expected behavior

`tests/programs/bug_array_struct_field_mutation.con` prints `99` and
`0` on successive lines (the test exercises both the bug shape and a
control case).

## Regression command

`make test-wrong-code` (registered as runtime case with multi-line
expected output).

## Historical context

Pre-corpus bug. Header comment in the repro file is the operative
description. The fix landed in `Concrete/Lower.lean`'s
arrayIndexAssign / array-from-field path; precise commit not pinned.
