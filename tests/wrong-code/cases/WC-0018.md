# WC-0018 — string_push_char + string_append in-place mutation

**Category:** miscompile
**Status:** fixed
**Discovered:** historical Bug 011

## Symptom

`string_push_char` and `string_append` did not mutate the receiver
in place — at landing they either no-op'd or mutated a
materialized copy, so a loop that built a string by appending
characters produced an empty result.

## Expected behavior

`tests/programs/bug_string_building.con` returns `0` (the test
builds a string by appending in a loop, then reads back to verify
the contents).

## Regression command

`make test-wrong-code`.

## Historical context

Pre-corpus Bug 011. Fix was in the in-place mutation path of the
String stdlib intrinsics — `string_push_char` and `string_append`
both take `&mut String` and must mutate the heap-allocated
representation through the borrow without materializing.
